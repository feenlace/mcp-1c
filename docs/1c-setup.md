# Установка 1С и настройка HTTP-сервиса для MCP

Инструкция для тех, у кого ещё не установлена 1С:Предприятие.

> Если 1С уже установлена - переходите сразу к [быстрому старту](getting-started.md).

## Шаг 1. Скачать и установить 1С

**Учебная версия (бесплатно):**

1. Перейти на https://uc1.1c.ru/uchebnaya-versiya-1s/
2. Зарегистрироваться и скачать дистрибутив
3. Запустить установщик, следовать инструкциям (все настройки по умолчанию)

> Учебная версия полностью функциональна, но работает только с файловыми базами и ограничена одним пользователем. Для наших целей этого достаточно.

## Шаг 2. Создать информационную базу

1. Запустить **1С:Предприятие**
2. Нажать **"Добавить"** -> **"Создание новой информационной базы"**
3. Выбрать шаблон (например, **"Бухгалтерия предприятия (учебная)"**) или создать пустую базу
4. Указать имя базы и путь для хранения файлов
5. Нажать **"Готово"**

## Шаг 3. Установить расширение mcp-1c

```bash
# Windows
mcp-1c --install "C:\путь\к\базе"

# macOS
mcp-1c --install ~/Documents/InfoBase

# Linux
mcp-1c --install /path/to/infobase
```

Программа сама найдет платформу, загрузит расширение и обновит конфигурацию БД.

> Если платформа установлена в нестандартную папку:
> `mcp-1c --install "путь" --platform "/custom/path/to/1cv8"`
>
> Если версия платформы не определяется автоматически (нестандартный путь без номера версии), укажите её явно:
> `mcp-1c --install "путь" --platform "/custom/path/to/1cv8" --platform-version 8.3.13`

> Если в базе настроена аутентификация для Конфигуратора (DESIGNER):
> `mcp-1c --install "путь" --db-user Admin --db-password пароль`

> **Клиент-серверная база (MS SQL, PostgreSQL).** Если база зарегистрирована на сервере 1С, добавьте флаг `--server` и укажите строку подключения в формате `сервер\имя_базы`:
> ```bash
> mcp-1c --install "srv-1c\buh_prod" --server --db-user Admin --db-password pass
> ```

### Установка через Конфигуратор (для RDP и серверов без CLI) {#manual-install}

Если нет доступа к командной строке (например, аренда 1С через RDP) — используйте готовый `.cfe` из релизов. Это самый простой путь.

**Вариант A. Готовый `.cfe` из Releases (рекомендуется)**

1. Скачайте `MCP_HTTPService.cfe` со страницы [Releases](https://github.com/feenlace/mcp-1c/releases/latest).
2. (Опционально) Проверьте контрольную сумму через `checksums.txt` рядом с файлом релиза:
   ```bash
   # macOS / Linux
   shasum -a 256 -c checksums.txt --ignore-missing
   # Windows (PowerShell)
   Get-FileHash MCP_HTTPService.cfe -Algorithm SHA256
   ```
3. Откройте базу в **Конфигураторе**.
4. Меню **Конфигурация -> Расширения конфигурации** (или панель «Расширения конфигурации»).
5. Нажмите **«Добавить»** (кнопка «+»), затем **«Добавить из файла...»** и выберите скачанный `MCP_HTTPService.cfe`.
6. Убедитесь, что флаг **«Активно»** установлен. «Безопасный режим» можно оставить по умолчанию.
7. Нажмите **F7** (Обновить конфигурацию базы данных) и подтвердите принятие изменений.
8. Перейдите в режим **1С:Предприятие** и проверьте доступность HTTP-сервиса (см. [Шаг 4](#шаг-4-запустить-http-сервис-1с)).

> **Совместимость.** `.cfe` собран на платформе **1С:Предприятие 8.3.14** (минимально поддерживаемая версия) — загружается на 8.3.14 и выше.
>
> **Профили безопасности (8.3.21+).** Если в базе включены профили безопасности и расширение блокируется при загрузке, откройте свойства расширения в Конфигураторе и снимите флаг **«Защита от опасных действий»**, затем повторите F7.

**Вариант B. Собрать `.cfe` самостоятельно**

Если есть локально установленная платформа 1С — соберите `.cfe` из XML-исходников через скрипт:

```bash
# macOS / Linux
./scripts/build-extension.sh ~/Documents/InfoBase

# Windows
scripts\build-extension.cmd C:\Users\User\Documents\InfoBase
```

Скрипт принимает путь к временной файловой ИБ (создайте пустую при необходимости) и опциональный путь для выгрузки `.cfe`. Подробнее — в [scripts/build-extension.sh](../scripts/build-extension.sh). Дальше — те же шаги 3-8 из варианта A.

**Вариант C. Загрузка XML-исходников через `LoadConfigFromFiles`**

Только для разработчиков, у которых есть Конфигуратор и нужно работать с исходниками без сборки `.cfe`:

1. Откройте базу в **Конфигураторе**.
2. Меню **Конфигурация -> Расширения конфигурации**, создайте новое расширение `MCP_HTTPService`.
3. С активным расширением: **Конфигурация -> Загрузить конфигурацию из файлов...**, выберите папку `extension/src/`.
4. Нажмите **F7**.

## Шаг 4. Запустить HTTP-сервис 1С

Чтобы mcp-1c мог обращаться к базе, нужно опубликовать HTTP-сервис 1С. Есть два способа:

### Способ 1. Публикация через Apache или IIS (рекомендуется)

Стандартный способ публикации HTTP-сервисов 1С. Работает на **Windows** (IIS или Apache) и **Linux** (Apache или ibsrv). Подходит как для разработки, так и для постоянной работы.

1. Откройте базу в **Конфигураторе от администратора**
2. **Администрирование -> Публикация на веб-сервере**
3. Укажите имя (например `base`), отметьте `MCPService`
4. Нажмите **Опубликовать**

URL в этом случае: `http://localhost/base/hs/mcp-1c`

> Подробнее о публикации через веб-сервер — в [документации 1С](https://its.1c.ru/db/v8317doc#bookmark:dev:TI000001540).

### Способ 2. Встроенный HTTP-сервер /HTTPPort (только Windows, для быстрого запуска)

Параметр `/HTTPPort` запускает встроенный HTTP-сервер прямо в процессе 1С. Это **недокументированный** параметр платформы, удобный для быстрой проверки и разработки. Для постоянной работы рекомендуется публикация через Apache или IIS (способ 1).

> **Ограничения:** `/HTTPPort` работает **только на Windows**. На Linux и macOS этот параметр недоступен.

**Windows (коммерческая):**
```cmd
"C:\Program Files\1cv8\8.3.XX.XXXX\bin\1cv8.exe" ENTERPRISE /F "C:\путь\к\базе" /HTTPPort 8080
```

**Windows (учебная):**
```cmd
"C:\Program Files\1cv8t\8.5.XX.XXXX\bin\1cv8t.exe" ENTERPRISE /F "C:\путь\к\базе" /HTTPPort 8080
```

> Замените `XX.XXXX` на вашу версию платформы.

Окно 1С откроется и будет работать как HTTP-сервер. **Не закрывайте его**.

### Платформенные ограничения

**macOS:** Платформа 1С [не поддерживает HTTP-сервисы на macOS](https://www.online-ufa.ru/content/articles/limit-1c-for-macos/). Используйте Windows (можно в виртуальной машине, например UTM). Автоустановка расширения (`--install`) работает на macOS, но для запуска HTTP-сервиса нужна Windows.

**Linux:** Используйте публикацию через Apache или автономный сервер ibsrv (способ 1). Встроенный HTTP-сервер (`/HTTPPort`) на Linux недоступен.

### Проверка

```bash
# Если публикация через Apache/IIS (способ 1):
curl http://localhost/base/hs/mcp-1c/version

# Если встроенный HTTP-сервер /HTTPPort (способ 2):
curl http://localhost:8080/hs/mcp-1c/version
```

Ожидаемый ответ: `{"version":"0.4.1"}`

## Устранение неполадок

**"1C platform not found" при --install**
Платформа не найдена в стандартных папках. Укажите путь явно через `--platform`. Если версия не определяется из пути, добавьте `--platform-version 8.3.13` (укажите вашу версию). Минимальная поддерживаемая версия: 8.3.10.

**"address already in use"**
Порт занят. Используйте другой: `/HTTPPort 9090` и обновите URL в конфиге AI-клиента.

**"404" при обращении к /hs/mcp-1c/...**
- Расширение не установлено или не активно. Проверьте: Конфигуратор -> Расширения -> галочка "Активно" напротив MCP_HTTPService.
- Конфигурация БД не обновлена. Нажмите F7 в Конфигураторе.

**"connection refused"**
HTTP-сервис 1С не запущен. Если используете `/HTTPPort` — убедитесь, что окно 1С открыто. Если публикацию через Apache/IIS — проверьте, что веб-сервер запущен и публикация активна.
